## About

DiagNet is a web-based network testing and validation platform built as a
diploma project at [HTL Rennweg](https://htlrennweg.at) in partnership with
[CANCOM Austria AG](https://cancom.at).

Instead of running tests manually through a terminal, DiagNet gives teams a
single web interface to manage devices, define tests, execute them in batches,
and track results over time. No command line needed.

---

## What It Does

{{< timeline >}}

{{< timelineItem icon="globe" header="Device Inventory" >}}
Add your routers and switches once. Credentials are stored with Fernet
encryption. DiagNet handles the connections so your tests don't have to.
{{< /timelineItem >}}

{{< timelineItem icon="wand-magic-sparkles" header="37+ Built-in Test Templates" >}}
Ping, BGP, OSPF, EIGRP, HSRP, IPSec, NAT, port security, AAA audits, and more.
All ready to use out of the box. Group tests together and run them in one go.
{{< /timelineItem >}}

{{< timelineItem icon="eye" header="Result History & Comparison" >}}
Every test run is stored. Compare results across attempts to catch network
drift, spot regressions, and prove stability over time.
{{< /timelineItem >}}

{{< timelineItem icon="lock" header="Multi-user Access Control" >}}
Role-based permissions let you control who can run tests, manage devices, or
administer the system.
{{< /timelineItem >}}

{{< /timeline >}}
